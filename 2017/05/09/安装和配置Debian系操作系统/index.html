其实我很早就尝试过Debian的，由于自己的当时还是个菜鸟，需要自己各种配置之后，中文才能正常显示，装完没多久就删掉了，Ubuntu提供了开箱即用的环境，我就使用了Ubuntu很长一段时间，但是我这种觉得无聊就会重装电脑的人，出来了新版本的系统就会跟着升级的，可笔记本HP-5310M的配置已经落后了，用官方的Live CD直接就启动不了，后来发现Debian系的Live CD都不能直接引导。转投了Arch Linux，Arch Linux的软件包太新了，天天都有一大堆的更新，经常更新完就挂掉了。不能因为新版本软件包里一个个新的Bug而天天无休止的折腾，就开始不太更新软件包，然后突然间有一天心血来潮了，更新一下，结果又挂掉了，心太累了！于是开始重新关注到Debian这个发行版，主要是看到Deepin Linux基于Debian滚动更新这条消息，让我觉得Debian是比Ubuntu更加靠谱的发行版本。

事实证明我的选择是对的，使用Debian的Stable之后，系统就没有因为更新挂掉过了。另外Deepin Linux对新手来说确实是一个不错的版本，对中文的支持是最好的，界面也漂亮，可是我已经过了需要用花哨的界面来装Cool的年龄，我需要一个省电省心的发行版本。不喜欢折腾的话，笔记本上我推荐使用Gnome作为桌面，另外Android手机和电纸书上我使用的是i3。

以前看到有人说高手都使用Arch Linux，我自己内心是经常自诩为高手的，所以就上当了。Arch Linux的理念是好的，也在折腾Arch Linux过程当中学到了很多东西，在最近的折腾道路上发现，Debian系的使用人群是比较大的，你的一些想法已经有人为你踩过坑了，google一下可能就有答案了。而使用Arch Linux作为关键词，可能会过滤掉这些答案。

当使用Debian Live CD无法正常引导你的笔记本的话，可以尝试使用其他发行版的Live CD，因为有Arch Linux的使用经验，所以我会结合Arch Linux的一些工具来帮助安装。如果你跟我一样，碰到了老笔记本无法正常引导启动，可以尝试禁用Grub启动器的背景图片或者使用Arch Linux Live CD，Arch Linux的软件仓库当中也有debootstrap软件包的。

搜狗输入法For Linux切换顿号和反斜杠的快捷键是ctrl+和ctrl+.。

2.3 亮度

Deepin Linux中显示器太亮，需要小于０.1的自定义亮度 gsettings set com.deepin.dde.display brightness {"eDP-1":0.08}

王勇离开Deepin Linux后，出现过一次比较严重的更新带来的故障，中文输入法完全无法输入，折腾我一天才找到问题所在。是我给老婆的电脑上安装的Deepin Linux，没有用上几天就碰到一个让普通用户绝望的巨大Bug，差点就想把Linux分区删掉了，老婆大人好好地用Windows不好，偏要给自己挖个坑。

2.4 远程端口映射

为什么要安利老婆使用Linux，因为Linux有太多好用的工具了，在出现故障的时候，我可以随时远程查障，不需要依赖远程协助还需要对方登录IM工具确认一下。这个方案是需要一个有公网IP的主机转发数据的，利用ssh的端口转发功能就可以将本地sshd服务端口映射到远程主机上。

Gnome 3虽然好用，但是占用的资源要多一些，曾经试过录屏出现一些问题，用ffmpg无法录屏，到现在也没有找到解决方案，据说是Gnome 3独特的图形加速驱动导致的，用基于Gnome 2的xfce4就一切都正常了。有一次我想保存一些做得非常好的视频素材，用Video Hunter下载后，发现视频做了加密处理，但是在线播放是没有问题，我就把浏览器全屏，分离控制台到另外一个显示器上，注入一些代码控制ffpmg来自动控制完成录屏。
